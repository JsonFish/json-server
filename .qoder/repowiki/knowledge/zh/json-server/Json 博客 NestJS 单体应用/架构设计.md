- 根入口 src/main.ts 通过 NestFactory.create 创建 NestExpressApplication，集中配置 express-session、全局 ValidationPipe、Swagger 文档以及 HTTP 异常过滤器。
- src/app.module.ts 作为唯一装配点：通过 TypeOrmModule.forRoot 注入 mysqlConfig 建立数据库连接；imports 聚合 UserModule、AuthModule、ArticleModule 三个业务模块；providers 中以 APP_FILTER / APP_INTERCEPTOR / APP_GUARD token 全局注册 AllExceptionFilter、TransformInterceptor、AuthGuard，使所有子模块共享同一套横切能力。
- 子模块（api/user、api/auth、api/article）各自维护独立的 Module/Controller/Service/Entity/DTO，仅通过 app.module.ts 暴露的依赖（TypeORM、全局 Guard/Interceptor）进行协作，彼此不直接 import。
- 构建由 nest-cli.json 驱动，tsconfig 路径映射（@/core/*、@/config/* 等）被各子模块复用，形成“单进程多模块”的装配契约。
