模块按 NestJS 扩展点分层组织：
- `src/core/filter/`：实现两个 `ExceptionFilter`——`HttpExceptionFilter` 仅捕获业务 `HttpException`（将 400 状态码归一为 200），`AllExceptionFilter` 通过 `@Catch()` 兜底所有未处理异常，两者均输出 `{ code, message, data: { query/body/params/method/url } }` 的统一错误体。
- `src/core/guard/`：`AuthGuard` 结合 `Reflector` 读取 `IS_PUBLIC_KEY` 元数据以跳过 `/auth/refresh` 之外的路由鉴权；从 `Authorization: Bearer <token>` 头解析 token，并根据 URL 区分使用 `accessSecretKey` 或 `refreshSecretKey` 进行 `verifyAsync`，校验通过后把 payload 挂到 `request.user`。
- `src/core/interceptor/`：`TransformInterceptor` 在成功路径上把任意返回值包装为 `{ code: 200, data, message: 'success' }` 的统一成功体，与异常过滤器共同构成全链路响应规范。
- `src/common/dto/pagination.dto.ts`：基于 `class-validator` + `class-transformer` 声明 `page`/`pageSize` 分页参数，默认值分别为 1 和 20。
依赖方向单向向外：core 层消费 common 中的 DTO，guard 依赖 `@nestjs/jwt` 与 `@/config/jwt.config`，filter/interceptor 不反向依赖业务模块。
