模块由三个层次构成：
- 进程入口 `src/main.ts`：通过 `NestFactory.create<NestExpressApplication>` 创建 Express 应用，集中注册 `express-session`、`ValidationPipe`、`HttpExceptionFilter`、Swagger (`/api-docs`) 等横切能力，并监听 `process.env.PORT`。
- 根模块 `src/app.module.ts`：作为 Nest 容器根节点，使用 `TypeOrmModule.forRoot(mysqlConfig)` 注入数据库连接，聚合 `UserModule`、`AuthModule`、`ArticleModule` 等业务模块，并通过 `APP_FILTER` / `APP_INTERCEPTOR` / `APP_GUARD` 令牌注册全局异常过滤、响应转换和鉴权守卫。
- 配置层 `src/config/*.config.ts`：每个外部依赖一个默认导出的纯对象（`mysql.config.ts` 类型为 `TypeOrmModuleOptions`，`jwt.config.ts` 提供 access/refresh token 密钥，`github.config.ts` 提供 OAuth client_id/client_secret），以常量形式被上层模块直接 import，尚未接入 `@nestjs/config` 的运行时环境变量机制。
依赖方向单向：main → AppModule → config/*；业务模块仅依赖 config 提供的对象，不反向依赖 main。
