- 实体统一使用 TypeORM 装饰器声明列名映射（如 `is_top`、`create_time`），并在所有查询条件中显式过滤 `is_deleted: 0` 实现软删除。
- DTO 通过 class-validator 装饰器定义必填/类型约束，数值型查询参数配合 `@Type(() => Number)` 进行类型转换。
- 控制器方法体仅做参数解构与转发，业务逻辑全部下沉到 Service，Controller 不直接访问 Repository。
- 对外响应统一包裹结构化对象（如 `{ articleList, total }`、`{ ...article, tags }`），避免直接返回裸实体。
