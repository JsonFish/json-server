采用 NestJS 标准分层：`article.module.ts` 通过 `TypeOrmModule.forFeature([Article, Tag])` 将两个实体注册为本地仓储；`article.controller.ts` 暴露 `/article` 路由，使用 `@Public()` 装饰器标记公开接口（来自 `@/core/guard/public.decorator`），其余接口受全局鉴权守卫保护；`article.service.ts` 通过 `@InjectRepository(Article)` / `Tag` 注入 TypeORM Repository 完成数据访问。数据模型方面，`entities/article.entity.ts` 与 `entities/tag.entity.ts` 均为扁平表结构，标签关系以逗号分隔字符串 `tagIds` 存储而非多对多关联，服务层在 `findAll` 中按 `tagIds.split(',')` 再查 `tag` 表回填 `tags` 字段。DTO 位于 `dto/article.dto.ts`，基于 `class-validator` + `class-transformer` 做入参校验，并复用 `@/common/dto/pagination.dto` 实现分页。依赖方向单向：controller → service → repository → database，模块不向外暴露内部类。
