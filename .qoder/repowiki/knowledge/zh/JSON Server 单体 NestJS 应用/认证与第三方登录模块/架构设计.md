采用 NestJS 标准分层：`auth.controller.ts` 暴露 `/auth/refresh`（需鉴权）与 `/auth/github`（通过 `@Public()` 装饰器跳过守卫）两个端点；`auth.service.ts` 封装业务逻辑，依赖 `UserService` 完成用户查找/注册与登录埋点更新，使用 `axios` 直接调用 GitHub OAuth 与用户信息 API，并通过 `JwtService` 结合 `@/config/jwt.config` 中的 access/refresh 密钥签发双令牌；`entities/email-code.entity.ts` 为 TypeORM 实体，`dto/auth.dto.ts` 提供 class-validator 校验规则。`AuthModule` 在 `imports` 中注入全局 `JwtModule`、`UserModule` 及 `TypeOrmModule.forFeature([EmailCode])`，对外仅暴露控制器与提供者，内部依赖方向单向指向 `user` 模块与共享配置/工具。
